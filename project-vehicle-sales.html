Automotive Analytics

Vehicle Sales & Pricing Analysis

Role: End-to-end data analyst  •  Tools: SQL (Oracle), Excel, Visualization tools

Overview

This project explores a large dataset of vehicle sales and prices to understand how factors like make, model, body type, mileage, condition, and state influence selling price, and how those prices compare to reference market values.

Dataset & Schema

The core table includes fields such as:

  • year, make, model, trim, body
  • transmission, state, condition, odometer
  • mmr (market reference value), sellingprice, saledate

Data was cleaned to remove invalid rows, fix formatting issues, and ensure that numeric and date fields were stored in appropriate data types.

Process

  1. Imported a raw CSV into Oracle and created a cleaned table.
  2. Standardized date formats and removed rows with clearly invalid values.
  3. Created derived fields (e.g., price vs. MMR ratio, mileage buckets, condition buckets).
  4. Aggregated results by make, model, body type, and state.
  5. Visualized the distribution of prices and identified outliers.
-- Example: price vs. MMR ratio by make and model
SELECT
    make,
    model,
    COUNT(*) AS vehicle_count,
    ROUND(AVG(sellingprice), 0) AS avg_sellingprice,
    ROUND(AVG(mmr), 0) AS avg_mmr,
    ROUND(AVG(sellingprice) / NULLIF(AVG(mmr), 0), 3) AS price_to_mmr_ratio
FROM car_sales_clean
GROUP BY make, model
HAVING COUNT(*) >= 50
ORDER BY price_to_mmr_ratio DESC;

Key Insights

  • Certain brands and models consistently sell above their reference market value, suggesting strong demand or favorable perception.
  • High-mileage vehicles with good “condition” scores can still achieve solid prices when paired with strong brands or popular body types (e.g., SUVs).
  • Some states show systematically higher or lower price-to-MMR ratios, hinting at regional pricing differences.
